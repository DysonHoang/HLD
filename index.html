01 — Vấn Đề Cần Giải Quyết

Tại sao cần HLD?

Giả sử có cây N đỉnh, mỗi đỉnh có giá trị. Bài toán yêu cầu:

❓

Truy vấn đường đi

Tính max/min/sum trên đường đi từ đỉnh u đến đỉnh v.

✏️

Cập nhật điểm

Cập nhật giá trị một đỉnh bất kỳ trên cây.

🔁

Nhiều truy vấn

Có thể lên đến 10⁵ truy vấn — cần mỗi truy vấn chạy nhanh.

🐢

Cách ngây thơ O(N)

Duyệt từng đỉnh trên đường đi → O(N) / query → quá chậm!

💡

Giải pháp: Heavy-Light Decomposition

HLD biến cây thành một dãy tuyến tính (flatten), rồi dùng Segment Tree để trả lời mỗi query trong O(log²N). Với N=10⁵ → ~17² ≈ 289 thao tác thay vì 10⁵.

02 — Khái Niệm Cốt Lõi

Heavy child & Light child

Với mỗi đỉnh u, định nghĩa sz[u] = kích thước cây con gốc u. Trong số các con của u, đứa con có sz lớn nhất gọi là heavy child, phần còn lại là light child.

🏋️

Heavy edge (cạnh nặng)

Cạnh nối từ đỉnh cha đến heavy child. Tạo thành các chuỗi dài xuyên suốt cây.

🍃

Light edge (cạnh nhẹ)

Cạnh nối đến light child. Khi đi qua light edge, kích thước cây con ít nhất nhân đôi.

⛓️

Heavy chain (chuỗi nặng)

Nối các heavy edge liên tiếp thành một chuỗi từ một đỉnh xuống đến lá.

🔑

Tính chất quan trọng

Từ gốc đến bất kỳ lá nào, ta đi qua tối đa O(log N) light edge!

📐

Chứng minh: tối đa O(log N) chuỗi trên đường đi

Khi đi qua light edge từ u đến con v, ta có sz[v] ≤ sz[u]/2 (vì v không phải heavy child). Kích thước giảm ít nhất 2 lần mỗi lần đổi chuỗi → tối đa log₂N lần đổi.

04 — Flatten Cây về Mảng Tuyến Tính

DFS tô số vị trí (pos[])

DFS lần 2 gán cho mỗi đỉnh một vị trí pos[u] trong mảng tuyến tính. Quy tắc: luôn đi theo heavy child trước — điều này đảm bảo mỗi heavy chain chiếm một đoạn liên tiếp trong mảng.

Tại sao liên tiếp lại quan trọng?

Khi các đỉnh cùng chain nằm liền kề trong mảng, ta có thể dùng Segment Tree để query/update cả đoạn chỉ trong O(log N). Đây là điểm cốt lõi của HLD!

05 — Truy Vấn Đường Đi u→v

Query trên path

Để query path(u, v): di chuyển cả hai đỉnh lên cùng một chain bằng cách nhảy light edge (lấy head của chain rồi nhảy lên cha), sau đó query đoạn cuối cùng trong cùng chain.

1

Nếu u và v khác chain: nhảy chain nào có head sâu hơn

So sánh depth[head[u]] và depth[head[v]]. Đỉnh nào có head sâu hơn thì query từ pos[head] đến pos[u] trên Segment Tree, rồi nhảy lên parent[head[u]].

2

Khi u và v cùng chain: query đoạn cuối

Khi head[u] == head[v], query đoạn [pos[u], pos[v]] (hoặc [pos[v], pos[u]]) trực tiếp trên Segment Tree.

3

Tổng hợp kết quả

Kết quả cuối = tổng hợp tất cả các đoạn query được (sum, max, min...). Số lần nhảy chain ≤ O(log N), mỗi query Seg Tree tốn O(log N) → tổng O(log²N).

// ── Segment Tree — hỗ trợ point update, range query ── const int MAXN = 2e5 + 5; int seg[MAXN * 4]; // seg[i] = giá trị node i của Seg Tree // Build từ mảng val[] (đã flatten từ cây) void build(int node, int l, int r, int* val) { if (l == r) { seg[node] = val[l]; return; } int mid = (l + r) / 2; build(2*node, l, mid, val); build(2*node+1, mid+1, r, val); seg[node] = seg[2*node] + seg[2*node+1]; // sum } // Update vị trí idx với giá trị newval void update(int node, int l, int r, int idx, int val) { if (l == r) { seg[node] = val; return; } int mid = (l + r) / 2; if (idx <= mid) update(2*node, l, mid, idx, val); else update(2*node+1, mid+1, r, idx, val); seg[node] = seg[2*node] + seg[2*node+1]; } // Query sum đoạn [ql, qr] int query(int node, int l, int r, int ql, int qr) { if (qr < l || r < ql) return 0; if (ql <= l && r <= qr) return seg[node]; int mid = (l + r) / 2; return query(2*node, l, mid, ql, qr) + query(2*node+1, mid+1, r, ql, qr); }
// ── Các mảng cần thiết cho HLD ────────────────── int sz[MAXN]; // kích thước cây con int depth[MAXN]; // độ sâu int parent[MAXN];// cha int heavy[MAXN]; // heavy child (-1 nếu là lá) int head[MAXN]; // head của chain chứa đỉnh này int pos[MAXN]; // vị trí trong mảng flatten int posVal[MAXN];// giá trị theo thứ tự pos int cur_pos; // bộ đếm pos hiện tại vector<int> adj[MAXN]; // danh sách kề int val[MAXN]; // giá trị ban đầu của đỉnh // DFS 1: tính sz[], depth[], parent[], heavy[] int dfs1(int u, int p, int d) { parent[u] = p; depth[u] = d; sz[u] = 1; heavy[u] = -1; int maxSz = 0; for (int v : adj[u]) { if (v == p) continue; // bỏ qua cha sz[u] += dfs1(v, u, d+1); // cập nhật heavy child if (sz[v] > maxSz) { maxSz = sz[v]; heavy[u] = v; } } return sz[u]; } // DFS 2: gán pos[], head[], xây posVal[] void dfs2(int u, int h) { head[u] = h; pos[u] = cur_pos; posVal[cur_pos] = val[u]; cur_pos++; // Đi heavy child trước (để chain liên tiếp) if (heavy[u] != -1) dfs2(heavy[u], h); // giữ nguyên head for (int v : adj[u]) { if (v == parent[u]) continue; if (v == heavy[u]) continue; // light child → chain mới, head = v dfs2(v, v); } } // Khởi tạo HLD void buildHLD(int root, int n) { dfs1(root, -1, 0); cur_pos = 0; dfs2(root, root); build(1, 0, n-1, posVal); // build Seg Tree }
// ── Query sum trên đường đi u→v ───────────────── int queryPath(int u, int v, int n) { int result = 0; // Lặp cho đến khi u và v cùng chain while (head[u] != head[v]) { // Đảm bảo u là đỉnh có head sâu hơn if (depth[head[u]] < depth[head[v]]) swap(u, v); // Query từ head[u] đến u trên Seg Tree result += query(1, 0, n-1, pos[head[u]], pos[u]); // Nhảy lên cha của head[u] (đổi chain) u = parent[head[u]]; } // u và v cùng chain — query đoạn cuối if (depth[u] > depth[v]) swap(u, v); result += query(1, 0, n-1, pos[u], pos[v]); return result; } // ── Update giá trị đỉnh u ──────────────────────── void updateNode(int u, int newVal, int n) { val[u] = newVal; update(1, 0, n-1, pos[u], newVal); }
// ── Main: Ví dụ sử dụng ───────────────────────── #include <bits/stdc++.h> using namespace std; int main() { ios_base::sync_with_stdio(false); cin.tie(nullptr); int n, q; cin >> n >> q; // số đỉnh, số truy vấn for (int i = 1; i <= n; i++) cin >> val[i]; for (int i = 0; i < n-1; i++) { int u, v; cin >> u >> v; adj[u].push_back(v); adj[v].push_back(u); } buildHLD(1, n); // gốc = 1 while (q--) { int type, u, v; cin >> type >> u >> v; if (type == 1) { // Query sum path(u, v) cout << queryPath(u, v, n) << "\n"; } else { // Update đỉnh u thành giá trị v updateNode(u, v, n); } } return 0; }
07 — Độ Phức Tạp

Phân tích hiệu suất

Build HLD
O(N log N)
2 lần DFS + build Seg Tree
Query path
O(log²N)
O(log N) chains × O(log N) Seg Tree
Update node
O(log N)
Point update trên Seg Tree

Với N = Q = 10⁵: Build tốn ~1.7×10⁶ ops, mỗi query ~17² = 289 ops × 10⁵ query = ~2.9×10⁷ ops. Hoàn toàn đủ nhanh trong giới hạn thời gian 1-2 giây.

⚡

Biến thể nâng cao

Thay Segment Tree bằng BIT (Fenwick Tree) để constant factor nhỏ hơn. Hoặc dùng Lazy Segment Tree cho range update. Tổng thể vẫn O(log²N).

08 — Lưu Ý & Ứng Dụng

Những điều cần nhớ

⚠️

Cạnh vs Đỉnh

Code trên xử lý giá trị trên đỉnh. Nếu bài yêu cầu giá trị trên cạnh, cần ánh xạ giá trị cạnh (u, v) vào đỉnh con (đỉnh sâu hơn), và khi query cùng chain cần loại trừ LCA (pos[lca]+1).

⚠️

Indexing 0-based vs 1-based

Phải nhất quán trong toàn bộ code. Ví dụ trên dùng đỉnh 1-indexed nhưng pos[] 0-indexed. Dễ nhầm lẫn — hãy kiểm tra kỹ boundary của Segment Tree.

💡

Tìm LCA bằng HLD

HLD cũng cho phép tìm LCA (Lowest Common Ancestor) trong O(log N): trong quá trình query path, khi head[u] == head[v], đỉnh nông hơn chính là LCA.

🏆

Bài toán điển hình dùng HLD

SPOJ QTREE (Query on a tree), Codeforces 342E (xquery trên cây), bài toán cập nhật/query tổng/max trên đường đi trong cây — đây là template chuẩn trong competitive programming.
